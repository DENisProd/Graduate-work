## Развёртывание

```bash
# Разработка
pnpm install
cp .env.example .env.local   # заполнить AUTH_* и NEXT_PUBLIC_*
pnpm dev                     # http://localhost:3000
```

```bash
# Docker (весь стек из корня монорепозитория)
cp .env.cloud.example .env
docker compose build frontend
docker compose up -d
```

### Переменные окружения

| Переменная | Описание |
|---|---|
| `AUTH_SECRET` | Секрет сессий — `npx auth secret` |
| `AUTH_KEYCLOAK_ID` | Client ID из Keycloak |
| `AUTH_KEYCLOAK_SECRET` | Client Secret из Keycloak |
| `AUTH_KEYCLOAK_ISSUER` | URL realm, например `http://localhost:8080/realms/smart-home` |
| `AUTH_URL` | Публичный URL этого приложения, например `http://localhost:3000` |
| `NEXT_PUBLIC_API_URL` | URL api-gateway (должен быть доступен из браузера) |
| `NEXT_PUBLIC_ACCESS_API_URL` | URL access-service (из браузера) |
| `NEXT_PUBLIC_PHYSICAL_DEVICES_API_URL` | URL scenario-service (из браузера) |

> `NEXT_PUBLIC_*` вшиваются в JS-бандл **при сборке**. При смене домена нужен повторный `pnpm build` или `docker compose build frontend`.

## Лендинг

**Маршрут:** `/`  
**Доступ:** публичный

Маркетинговая страница со GSAP-анимациями. Описывает возможности системы, архитектуру (облако + edge + клиент) и целевую аудиторию.

**Секции страницы:**
- Hero с mockup панели управления
- «Почему Domovoy» — сравнение проблем рынка с решением системы
- Ключевые возможности: гибридная работа, RBAC/ABAC-доступ, поддержка Zigbee и Modbus, мультиобъекты, реалтайм, AI
- Для кого: частный дом, офис, ритейл, склад
- Архитектура: три слоя — облако, edge, клиент
- Сценарии использования: подключение устройств, план этажа, работа оффлайн, синхронизация, AI-автоматизация, мультиобъект
- CTA → `/dashboard` и → `/admin`

## Аутентификация

### `/auth/signin`

**Доступ:** публичный

Страница входа. Единственная кнопка — «Войти через Keycloak», которая инициирует OAuth 2.0 redirect. После успешной аутентификации редиректит на `callbackUrl` (по умолчанию `/dashboard`).

**Что может пользователь:**
- Нажать «Войти» → перейти в Keycloak для ввода логина и пароля
- Вернуться на страницу, с которой пришёл, после входа

### `/auth/error`

**Доступ:** публичный

Отображает сообщение об ошибке аутентификации (неверный токен, сессия истекла, отказ Keycloak). Предлагает попробовать снова → `/auth/signin`.

## Дашборд пользователя

Все страницы раздела `/dashboard` требуют авторизации. Неавторизованный пользователь перенаправляется на `/auth/signin`.

---

### Главная — список домов (`/dashboard`)

Главный экран после входа.

**Что отображается:**
- Карточки всех домов текущего пользователя (сетка 1–3 колонки)
- Каждая карточка: название дома, адрес, роль пользователя в этом доме, количество устройств
- Поисковая строка для фильтрации по названию
- Сводная панель статистики: суммарное число устройств, активных сценариев, событий за 24 ч
- Лента последних событий управления: время, дом, устройство, действие, результат (SUCCESS / DENIED / ERROR)

**Что может пользователь:**
- Найти дом по названию через поиск
- Создать новый дом (кнопка «Создать дом» → модальное окно с названием и адресом)
- Отредактировать название/адрес дома через контекстное меню карточки
- Перейти в конкретный дом → `/dashboard/houses/[houseId]`

**Переходы:** карточка дома → `/dashboard/houses/[houseId]`, шапка → `/dashboard/settings`, `/profile`

---

### Обзор дома (виджеты) (`/dashboard/houses/[houseId]`)

Главный экран конкретного дома. Отображает настраиваемую виджет-панель с реальным состоянием устройств.

**Что отображается:**
- Сетка виджетов (react-grid-layout) — перетаскиваемые и изменяемые по размеру
- Виджеты обновляются в реальном времени через Socket.IO (`/zigbee`)
- Если панель пустая — подсказка с приглашением добавить первый виджет

**Типы виджетов:**

| Виджет | Назначение |
|---|---|
| `TELEMETRY_VALUE` | Числовое значение, бейдж или булев флаг из состояния устройства (температура, влажность, яркость…) |
| `DEVICE_STATUS` | Онлайн / оффлайн-статус устройства, время последнего сигнала |
| `CONTROL_BUTTON` | Кнопка, посылающая команду устройству (одноразовое действие) |
| `CONTROL_TOGGLE` | Переключатель с синхронизацией текущего состояния (вкл/выкл) |
| `SCENARIO_TRIGGER` | Кнопка ручного запуска сценария |
| `TEXT_LABEL` | Заголовок, подзаголовок или разделитель |
| `GAUGE_DIAL` | Аналоговый циферблат для числового значения |
| `CIRCULAR_PROGRESS` | Круговой прогресс-бар (например, заряд батареи) |
| `SLIDER_CONTROL` | Слайдер для установки числового значения (яркость, температура) |
| `DEVICE_HERO` | Крупная карточка устройства с иконкой и основными показателями |
| `MINI_LINE_CHART` | Линейный график истории телеметрии в реальном времени |

**Что может пользователь:**
- Перетаскивать и изменять размер виджетов (режим редактирования)
- Добавить виджет через «+» → выбрать тип из каталога → настроить (привязать к устройству, указать ключ payload, единицы измерения)
- Удалить виджет
- Управлять устройствами прямо с виджетов (кнопки, тоглы, слайдеры)
- Запустить сценарий с виджета `SCENARIO_TRIGGER`
- Сохранить раскладку (раскладка сохраняется в scenario-service привязанной к пользователю и дому)
- Переключаться между вкладками дома: Комнаты, Устройства, Сценарии, Участники, Роли, Настройки

---

### Комнаты (`/dashboard/houses/[houseId]/rooms`)

**Доступ:** все участники дома

**Что отображается:**
- Список комнат в виде таблицы: название, ID
- Кнопка «Открыть план» для перехода в редактор

**Что может пользователь:**
- Просмотреть список комнат
- Создать новую комнату (Owner / тот, кто может управлять устройствами) → модальное окно с названием
- Редактировать название комнаты
- Удалить комнату
- Перейти в планировщик помещений → `/dashboard/houses/[houseId]/room-planner`

---

### Планировщик помещений (`/dashboard/houses/[houseId]/room-planner`)

Визуальный редактор на базе Konva.js для создания плана этажа.

**Что отображается:**
- Канвас с планом помещений
- Боковая панель с инструментами и списком элементов

**Что может пользователь:**
- Загрузить изображение плана этажа как фон
- Нарисовать/разместить комнаты на плане
- Перетащить иконки устройств на план
- Изменить масштаб и перемещаться по плану (zoom/pan)
- Сохранить расположение элементов

---

### Устройства дома (`/dashboard/houses/[houseId]/devices`)

**Доступ:** все участники дома

**Что отображается:**
- Список физических устройств, зарегистрированных в этом доме
- Для каждого: название, статус онлайн/оффлайн, последнее значение ключевых показателей, комната
- Фильтр по комнате

**Что может пользователь:**
- Просмотреть состояние всех устройств
- Отфильтровать по комнате
- Перейти к детальной странице устройства → `/dashboard/houses/[houseId]/devices/[deviceId]`

---

### Карточка устройства (`/dashboard/houses/[houseId]/devices/[deviceId]`)

**Доступ:** участники дома с соответствующими правами

**Что отображается:**
- Название устройства, статус онлайн/оффлайн, время последнего сигнала
- Все текущие показатели устройства (из Zigbee/Modbus state)
- Панель управляющих действий (если у устройства есть функции с действиями)

**Что может пользователь:**
- Просматривать реальное состояние устройства (обновляется через Socket.IO)
- Отправить команду устройству (если есть право `execute`)
- Вернуться к списку устройств

---

### Сценарии (`/dashboard/houses/[houseId]/scenarios`)

**Доступ:** все участники; управление — Owner и те, у кого есть `canManageAutomations`

**Что отображается:**
- Список сценариев: название, статус (активен/неактивен), тип триггера, дата создания
- Фильтр по статусу и создателю
- Пагинация

**Что может пользователь:**
- Включить / выключить сценарий (тоглом)
- Перейти к редактированию → `/admin/access-control/houses/[houseId]/scenarios/[id]`
- Создать новый сценарий → `/admin/access-control/houses/[houseId]/scenarios/new`
- Удалить сценарий (с подтверждением)

---

### Участники (`/dashboard/houses/[houseId]/members`)

**Доступ:** все участники (просмотр); приглашение/удаление — Owner и Admin

**Что отображается:**
- Список участников: отображаемое имя, роль в доме (Owner / Admin / Default / кастомная)
- Список активных приглашений: email, дата создания, срок действия, статус (PENDING / ACCEPTED / DECLINED / EXPIRED)

**Что может пользователь:**
- Просмотреть всех участников и их роли
- Нажать на участника → открыть боковую панель с деталями члена и его правами
- Удалить участника из дома (Owner / Admin)
- Отправить приглашение по email (Owner / Admin) → участник получит ссылку `/invite?token=…`
- Скопировать ссылку-приглашение
- Отозвать активное приглашение

---

### Роли (`/dashboard/houses/[houseId]/roles`)

**Доступ:** Owner и Admin (`canEditRoles`). Остальные участники перенаправляются на `/dashboard/houses/[houseId]`

**Что отображается:**
- Таблица ролей дома: системные (Owner, Admin, Default) и кастомные
- При выборе роли: список прав (`AccessRight`), список участников с этой ролью

**Что может пользователь:**
- Просмотреть набор прав каждой роли
- Создать кастомную роль с выбранным набором прав
- Редактировать права кастомной роли
- Удалить кастомную роль
- Назначить роль участнику (из вкладки «Участники»)

---

### Настройки дома (`/dashboard/houses/[houseId]/settings`)

**Доступ:** только Owner. Остальные перенаправляются на `/dashboard/houses/[houseId]`

**Что может пользователь:**
- Переименовать дом
- Изменить адрес
- Удалить дом (с подтверждением)

---

### Настройки профиля (`/dashboard/settings`)

**Что отображается:**
- Карточка настроек внешнего вида

**Что может пользователь:**
- Переключить тему: светлая / тёмная / системная (сохраняется в localStorage)
- Переключить язык интерфейса: RU / EN (сохраняется в localStorage)

## Панель администратора

Раздел `/admin` предназначен для управления системным каталогом и конфигурацией объектов. Требует авторизации; ролевые ограничения на уровне API.

---

### Главная администратора (`/admin`)

**Что отображается:**
- Плитки-ссылки на разделы: Устройства, Типы устройств, Категории устройств, Управление доступом, Modbus

**Что может пользователь:**
- Перейти в любой раздел администрирования

---

### Каталог — типы устройств (`/admin/device-types`)

Верхний уровень иерархии каталога: Освещение, Климат, Безопасность…

**Что отображается:**
- Таблица типов с кодом, названием (в текущем языке), статусом активности

**Что может пользователь:**
- Создать новый тип → модальное окно: код (UPPER_SNAKE), название и описание на RU и EN
- Редактировать существующий тип
- Деактивировать / активировать тип (soft delete — тип скрывается из выборок, но не удаляется)

---

### Каталог — категории (`/admin/device-categories`)

Второй уровень: каждая категория принадлежит типу. Например, тип «Освещение» → категория «Умная лампа».

**Что отображается:**
- Таблица категорий: код, название, родительский тип, статус

**Что может пользователь:**
- Создать категорию: выбрать тип из списка, задать код и переводы
- Редактировать категорию
- Деактивировать / активировать

---

### Каталог — устройства (`/admin/devices`)

Третий уровень: модели устройств (конкретные SKU).

**Что отображается:**
- Таблица устройств с фильтром по типу и категории, пагинацией
- Колонки: код, название (локаль), категория, статус, серийный номер

**Что может пользователь:**
- Создать устройство: выбрать категорию, задать код, серийный номер, версию ПО, переводы (RU/EN)
- Редактировать устройство
- Деактивировать / активировать
- Перейти к карточке устройства → `/admin/devices/[deviceId]`

---

### Карточка устройства каталога (`/admin/devices/[deviceId]`)

**Что отображается:**
- Шапка: название, код, категория
- Две вкладки: **Функции** и **Действия функций**

**Что может пользователь:**
- Редактировать метаданные устройства (кнопка «Редактировать» → модальное окно)
- Перейти на вкладку «Функции» для управления функциями
- Перейти на вкладку «Действия функций» для просмотра всех действий устройства
- Вернуться к списку → `/admin/devices`

---

### Функции устройства (`/admin/devices/[deviceId]/functions`)

Функция описывает, что умеет устройство: `switch`, `temperature`, `brightness`…

**Что отображается:**
- Таблица функций: код (lower_snake), переводы

### Дом (admin view) (`/admin/access-control/houses/[houseId]`)

Полный admin-обзор конкретного дома. Состоит из пяти вкладок, переключаемых через `?tab=`.

#### Вкладка «Комнаты»

#### Вкладка «Участники»

**Что может пользователь:**

- Удалить участника

#### Вкладка «Роли»

**Что отображается:** таблица ролей + детали выбранной роли

**Что может пользователь:**
- Создать / редактировать / удалить кастомную роль
- Назначить участнику роль
- Просмотреть список участников каждой роли

#### Вкладка «Устройства»

**Что отображается:**
- Список физических устройств дома с Zigbee-телеметрией
- Блок подключённых локальных серверов

**Что может пользователь:**

- Привязать физическое устройство к модели из каталога
- Просмотреть текущую телеметрию устройства
- Перейти к детальной странице устройства → `/admin/access-control/houses/[houseId]/devices/[deviceId]`
- Удалить физическое устройство из дома

#### Вкладка «Сценарии»

**Что отображается:** список сценариев дома с фильтрами по статусу и создателю

**Что может пользователь:**

### Конструктор сценариев (`/admin/access-control/houses/[houseId]/scenarios/builder`)

Визуальный граф-редактор для создания логики автоматизации. Построен на Konva.js.

**Интерфейс:**
- **Toolbar** — кнопки сохранить, отменить, очистить, переключить режим
- **CatalogPanel** (левая панель) — палитра доступных узлов, разбитых на группы
- **CanvasStage** (центр) — канвас, где расположены узлы и соединяющие их стрелки
- **PropertyPanel** (правая панель) — настройки выбранного узла

**Типы узлов:**

| Группа | Узлы |
|---|---|
| Триггеры | SCHEDULE (cron-расписание), MANUAL (ручной запуск), DEVICE_EVENT (событие устройства), WEBHOOK |
| Условия | ALWAYS, DEVICE_STATE, AND, OR, NOT |
| Действия | Команда устройству, DELAY, и другие |

**Что может пользователь:**
- Перетащить узел из каталога на канвас
- Соединить узлы стрелками (триггер → условие → действие)
- Настроить параметры узла в правой панели (выбрать устройство, задать cron, указать payload команды)
- Перемещать и удалять узлы
- Сохранить граф → сценарий обновится с новой `definition`

### Принятие приглашения (`/invite?token=…`)

**Доступ:** публичный (вход потребуется, если не авторизован)

Страница открывается по ссылке-приглашению, полученной от другого участника дома.

**Что отображается:**
- Название дома, в который приглашают
- Статус приглашения (PENDING / ACCEPTED / DECLINED / EXPIRED)
- Срок действия приглашения

**Что может пользователь:**
- Принять приглашение → автоматический редирект на `/dashboard` через 1.5 с
- Если не авторизован — ссылка на `/auth/signin?callbackUrl=/invite?token=…` с возвратом на эту страницу

**Примечание:** `/invite/[token]` — устаревший формат, автоматически перенаправляет на `/invite?token=…`

---

### Авторизация устройства (`/device-auth`)

**Доступ:** авторизованные пользователи

Страница подтверждения OAuth 2.0 Device Authorization Flow. Используется, когда локальный сервер запрашивает доступ к облачным API.

**Сценарий использования:**
1. Локальный сервер (Rust) инициирует Device Flow и получает `userCode`
2. Пользователь открывает `/device-auth?userCode=…&sessionId=…`
3. Страница отображает код подтверждения и запрашивает разрешение
4. Пользователь нажимает «Подтвердить» — локальный сервер получает токен доступа

**Что может пользователь:**
- Просмотреть запрошенные права доступа
- Подтвердить запрос (POST в access-service)
- Отклонить запрос

## Навигация между страницами

```
/ (лендинг)
├── → /auth/signin
├── → /dashboard
└── → /admin

/auth/signin → /dashboard (после успешного входа)

/dashboard
├── карточка дома → /dashboard/houses/[id]
├── кнопка настроек → /dashboard/settings
└── аватар/имя → /profile

/dashboard/houses/[id]   (вкладки в sidebar/header)
├── Обзор (виджеты)   → /dashboard/houses/[id]
├── Комнаты           → /dashboard/houses/[id]/rooms
│                          └── → /dashboard/houses/[id]/room-planner
├── Устройства        → /dashboard/houses/[id]/devices
│                          └── устройство → /dashboard/houses/[id]/devices/[deviceId]
├── Сценарии          → /dashboard/houses/[id]/scenarios
│                          └── редактировать → /admin/access-control/houses/[id]/scenarios/[id]
├── Участники         → /dashboard/houses/[id]/members
├── Роли              → /dashboard/houses/[id]/roles  (Owner/Admin)
└── Настройки         → /dashboard/houses/[id]/settings  (Owner)

/admin
├── Типы устройств     → /admin/device-types
├── Категории          → /admin/device-categories
├── Устройства         → /admin/devices
│                          └── устройство → /admin/devices/[id]
│                                ├── Функции → /admin/devices/[id]/functions
│                                │     └── → /admin/devices/[id]/functions/[fId]/actions
│                                └── Все действия → /admin/devices/[id]/function-actions
├── Управление доступом → /admin/access-control/houses
│                          └── дом → /admin/access-control/houses/[id]
│                                ├── tab=rooms    → rooms + room-planner
│                                ├── tab=members  → members
│                                ├── tab=roles    → roles
│                                ├── tab=devices  → physical devices
│                                └── tab=scenarios
│                                      ├── → /admin/access-control/houses/[id]/scenarios/new
│                                      ├── → /admin/access-control/houses/[id]/scenarios/[id]
│                                      └── → /admin/access-control/houses/[id]/scenarios/builder
└── Modbus             → /admin/modbus

/invite?token=…  →  /auth/signin (если не авторизован)  →  /dashboard (после принятия)
/device-auth     →  подтверждение  →  уведомление локальному серверу
```
